# User Context

This file contains user-specific context that customizes how the Background Research Agent (BEARY) conducts research.

## Audience

The research audience is a mid-level MLOps engineer who has prioritized breadth over depth. Expect technical literacy in most basic topics around MLOps, such as Docker, APIs, machine learning, etc. Provide more context for topics belonging to software engineering and data engineering such as backends, databases, networking, etc.

Assume that if the audience is asking a question, they have less expertise in this particular area.

For newer and cutting edge topics (within the last 3 months), assume that the audience has heard of the topic in passing, but does not know what it really is.

If the user shares the purpose of the research, use it to narrow down the search.

## Priorities

When researching, we should prioritize the following:
- Sources from documentation, troubleshooting from real users, code repositories, blogs, and literature. Company press releases and news articles are useful context, but do not prioritize their claims about what is technically feasible.
- Technical and practical solutions and knowledge, not theoretical ones.
